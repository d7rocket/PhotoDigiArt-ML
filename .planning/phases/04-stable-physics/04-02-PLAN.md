  <action>
    Create 7 WGSL compute shaders implementing the PBF algorithm (Macklin & Muller 2013). All shaders use workgroup_size(256).

    **Shared WGSL struct** (define in each shader that needs it, or use a common pattern):
    The SimParams struct matching the 128-byte uniform layout from Plan 01. Each shader declares its own bindings.

    **1. pbf_predict.wgsl** -- Entry point: `pbf_predict`
    Bindings: positions_in (read), predicted_out (write), home_positions (read), params (uniform)
    Algorithm:
    - Read current position and velocity from positions_in
    - Compute home attraction force: spring force toward home position (elastic tether). Strength = params.home_strength * breathing_mod * feature_strength(from home.w). Force = direction * strength * distance.
    - Add gravity force from params
    - Apply force clamping: clamp total force magnitude to params.max_force
    - Update velocity: v += dt * force
    - Apply velocity clamping: clamp velocity magnitude to params.max_velocity
    - Compute predicted position: x_pred = x + dt * v
    - Write predicted position to predicted_out
    - NOTE: Curl noise will be added in Plan 04 (organic motion). For now, just home + gravity.

    **2. pbf_hash_count.wgsl** -- Entry point: `hash_count`
    Bindings: predicted_positions (read), cell_counts (atomic storage), params (uniform)
    Algorithm:
    - Hash predicted position to cell index: floor((pos + 64) / cell_size), clamp to [0, 127], flat hash = x + y*128 + z*128*128
    - atomicAdd(&cell_counts[hash], 1u)

    **3. pbf_hash_scan.wgsl** -- Entry point: `prefix_sum_up`, `prefix_sum_down`
    Bindings: data (read_write storage)
    Algorithm: Tree-reduction (Blelloch-style) parallel prefix sum.
    - Up-sweep: reduce pairs, storing partial sums
    - Down-sweep: scatter partial sums back
    - Uses workgroup_size(256), processes 512 elements per workgroup
    - Two entry points for the two sweep phases
    - NOTE: For 2M cells, this needs hierarchical dispatch (block sums). The PBFSolver Python class handles multi-level dispatch.

    **4. pbf_hash_scatter.wgsl** -- Entry point: `hash_scatter`
    Bindings: predicted_positions (read), cell_offsets (atomic read_write), sorted_indices (write), params (uniform)
    Algorithm:
    - Same hash as count pass
    - offset = atomicAdd(&cell_offsets[hash], 1u)
    - sorted_indices[offset] = particle_index

    **5. pbf_density.wgsl** -- Entry point: `compute_density`
    Bindings: predicted_positions (read), cell_counts (read), cell_offsets (read), sorted_indices (read), lambda_out (write), params (uniform)
    Algorithm:
    - For each particle i, search 3x3x3 neighbor cells
    - For each neighbor j: accumulate density using poly6 kernel W(|xi - xj|, h)
    - Compute constraint: C_i = (rho_i / rest_density) - 1
    - Compute sum of squared gradients of C_i (spiky kernel gradient)
    - lambda_i = -C_i / (sum_grad_sq + epsilon_pbf)
    - NaN/Inf guard: if isnan or isinf, set lambda to 0.0

    Poly6 kernel: W(r, h) = (315 / (64 * pi * h^9)) * (h^2 - r^2)^3 for r < h, else 0
    Spiky gradient: grad_W(r, h) = -(45 / (pi * h^6)) * (h - r)^2 * direction

    **6. pbf_correct.wgsl** -- Entry point: `compute_correction`
    Bindings: predicted_positions (read_write), lambda_buf (read), cell_counts (read), cell_offsets (read), sorted_indices (read), delta_p (write), params (uniform)
    Algorithm:
    - For each particle i, search 3x3x3 neighbor cells
    - For each neighbor j:
      - s_corr = -k * (W(|xi-xj|, h) / W(delta_q * h, h))^n  (artificial pressure)
      - dp_i += (1/rest_density) * (lambda_i + lambda_j + s_corr) * grad_W_spiky(xi - xj, h)
    - Write dp_i to delta_p buffer
    - Apply correction: predicted_positions[i] += dp_i

    **7. pbf_finalize.wgsl** -- Entry point: `pbf_finalize`
    Bindings: positions_in (read), predicted_positions (read), positions_out (write), params (uniform)
    Algorithm:
    - v_new = (predicted[i] - original[i]) / dt
    - Clamp velocity magnitude to max_velocity
    - Write new position (from predicted) and new velocity to positions_out
    - NOTE: Vorticity confinement and XSPH will be added in Plan 04. For now, just velocity from displacement + clamping.

    **Critical implementation details (from research):**
    - All atomic buffers must be declared as array<atomic<u32>> in WGSL
    - cell_size MUST equal kernel_radius (h) -- read from params.cell_size
    - Grid size is 128^3, hardcoded as const in shaders
    - Use substantial epsilon (600.0) to prevent division by zero
    - Add NaN/Inf guards on lambda and delta_p computations

    **Pipeline build details:**
    - Each shader needs its own bind group layout matching its declared bindings
    - predict: 4 bindings (positions_in, predicted_out, home_positions, params)
    - hash_count: 3 bindings (predicted, cell_counts, params)
    - hash_scan: 1 binding (data buffer) -- dispatched multiple times
    - hash_scatter: 4 bindings (predicted, cell_offsets, sorted_indices, params)
    - density: 6 bindings (predicted, cell_counts, cell_offsets, sorted_indices, lambda_out, params)
    - correct: 7 bindings (predicted, lambda, cell_counts, cell_offsets, sorted_indices, delta_p, params)
    - finalize: 4 bindings (positions_in, predicted, positions_out, params)

    **Prefix sum strategy:**
    - 2M cells / 512 elements per workgroup = 4096 workgroups at level 1
    - 4096 block sums / 512 = 8 workgroups at level 2
    - Then fixup dispatches to add block sums back. Total: ~5 dispatches for prefix sum.
    - Create a temporary block_sums buffer during init.

    **Chunked dispatch:**
    - Use the same pattern as existing engine: 256K particles per dispatch, workgroup_size 256.
    - Each dispatch uses a separate command encoder submission for synchronization.

    **Bind group caching:**
    - Rebuild bind groups when buffer orientation changes (after swap).
    - Store bind groups for both "a" and "b" orientations.

    **Cell counts clearing:**
    - Before hash_count, zero the cell_counts buffer. Use device.queue.write_buffer with zero bytes for simplicity (8MB write for 2M cells is acceptable -- it is a one-time per-frame operation). Alternatively, create a small compute shader that writes zeros. CPU zero-write is simpler and acceptable for now.

    **Solver iterations:**
    - Read solver_iterations from params (cast to int). Loop density + correct dispatches that many times.
    - Each iteration re-dispatches density then correct. The predicted_positions buffer accumulates corrections across iterations.
