    **Update noise.wgsl** -- add a proper curl_noise_3d function:
    The existing noise.wgsl has perlin3d and fbm3d. Add a curl_noise_3d function that computes the curl of a 3D noise field to produce divergence-free flow (particles don't compress or expand, they flow).

    ```wgsl
    fn curl_noise_3d(pos: vec3<f32>, freq: f32, amp: f32, time: f32) -> vec3<f32> {
        let eps = 0.01;  // finite difference epsilon
        let p = pos * freq + vec3<f32>(time * 0.1, 0.0, 0.0);  // slow time evolution

        // Compute curl via finite differences of FBM noise:
        // curl_x = dNz/dy - dNy/dz
        // curl_y = dNx/dz - dNz/dx
        // curl_z = dNy/dx - dNx/dy

        // Sample noise at offset positions for each partial derivative
        let nx_dy = fbm3d(p + vec3(0.0, eps, 0.0), octaves) - fbm3d(p - vec3(0.0, eps, 0.0), octaves);
        // ... (6 samples total for 3 curl components)

        return normalize(curl) * amp;
    }
    ```

    Use low frequency (controlled by params.noise_frequency, default 0.5) and high amplitude (params.noise_amplitude, default 1.0) for the ocean-current feel per user decision.

    The curl noise function should accept an octaves parameter. Use 3 octaves for smooth, sweeping flows (not too noisy). The time offset creates slow evolution of the flow field.

    **Update pbf_predict.wgsl** -- integrate curl noise into the predict pass:

    The predict pass currently computes home_attraction + gravity. Add curl noise as an additional force:

    ```
    // After home attraction force:
    let curl_force = curl_noise_3d(pos, params.noise_frequency, params.noise_amplitude, params.time);

    // Photo-influenced motion: busy/complex photos get more turbulent flow
    // Use feature_strength from home_positions.w (high = edge/detail, low = flat)
    let motion_mod = mix(0.7, 1.3, feature_strength);  // flat areas: calmer, edges: more active
    let modulated_curl = curl_force * motion_mod;

    total_force += modulated_curl;
    ```

    The curl noise force is applied BEFORE velocity clamping. The home attraction (elastic tether) pulls back, curl noise pushes around -- the balance creates the "ocean currents around home position" aesthetic.

    **Breathing modulation in predict:**
    The breathing_mod value is already in params (computed per-frame by engine). Apply it to:
    - home_strength: effective_home = home_strength * breathing_mod (tighter on inhale, looser on exhale)
    - noise_amplitude: effective_amp = noise_amplitude * (2.0 - breathing_mod) (more motion on exhale, less on inhale -- complement)

    This creates the "inhale contracts/tightens, exhale relaxes/flows" cycle.

    **Shader composition:** pbf_predict.wgsl needs access to curl_noise_3d from noise.wgsl. Two approaches:
    1. Prepend noise.wgsl source before pbf_predict.wgsl when building the shader module (same pattern as existing _extract_noise_functions in engine.py)
    2. Or inline the curl function directly in pbf_predict.wgsl

    Use approach 1 -- update PBFSolver to use build_combined_shader("noise", "pbf_predict") when building the predict pipeline. This keeps noise functions shared and DRY.
