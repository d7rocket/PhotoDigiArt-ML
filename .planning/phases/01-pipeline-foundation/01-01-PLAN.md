    **Step 1: Project config** — Create `pyproject.toml` with:
    - `[project]` section: name="apollo7", python_requires=">=3.12"
    - Dependencies: PySide6>=6.8, pygfx>=0.16, rendercanvas>=2.6, wgpu>=0.31, opencv-python-headless>=4.10, Pillow>=11, onnxruntime-directml>=1.21, numpy>=2.1, scipy>=1.14, extcolors
    - Dev dependencies: pytest, pytest-timeout, ruff
    - `[tool.pytest.ini_options]`: testpaths=["tests"], timeout=30

    **Step 2: App bootstrap** — Create `apollo7/__main__.py` as entry point (`python -m apollo7`). Create `apollo7/app.py` with QApplication creation, theme application, MainWindow instantiation, and `app.exec()`.

    **Step 3: Config/settings** — Create `apollo7/config/settings.py` with defaults:
    - POINT_SIZE_DEFAULT = 2.0, POINT_SIZE_RANGE = (0.5, 10.0)
    - DEPTH_EXAGGERATION_DEFAULT = 4.0
    - BG_COLOR_TOP = "#1a1a1a", BG_COLOR_BOTTOM = "#0a0a0a"
    - ACCENT_COLOR = "#0078FF"
    - VIEWPORT_MIN_FPS = 30
    - LOD thresholds (use judgment for defaults)
    - WINDOW_SIZE = (1920, 1080), MIN_WINDOW_SIZE = (1280, 720)

    **Step 4: Dark theme** — Create `apollo7/gui/theme.py` with `load_theme_qss()` function returning a comprehensive QSS stylesheet. Follow the research example closely: dark backgrounds (#1a1a1a, #242424, #2d2d2d), electric blue accent (#0078FF), Segoe UI font, custom-styled QPushButton, QProgressBar, QScrollBar, QSplitter handles, QLabel, QGroupBox, QComboBox, QSlider. This must look like Maya x Unreal x modern SaaS — NOT stock Qt. Include hover/pressed states for interactive elements.

    **Step 5: Viewport widget** — Create `apollo7/gui/widgets/viewport_widget.py` with `ViewportWidget(QWidget)`:
    - Embed `QRenderWidget` from `rendercanvas.qt` with `update_mode="continuous"`
    - Create `gfx.WgpuRenderer`, `gfx.Scene`, `gfx.PerspectiveCamera(60)`
    - Add dark gradient background via `gfx.Background.from_color("#1a1a1a", "#0a0a0a")`
    - Set up `gfx.OrbitController` with the camera registered to the renderer
    - Implement `add_points(positions, colors, sizes)` method that creates `gfx.Points` with `gfx.PointsGaussianBlobMaterial(color_mode="vertex", size_mode="vertex")` and adds to scene
    - Implement `clear_points()` to remove all point objects from scene
    - Implement `auto_frame()` that calls `camera.show_object(scene)` to frame the sculpture
    - On init, generate test data: 10,000 random points in a sphere with random RGB colors and uniform size=2.0, add them to the scene and auto-frame. This proves the viewport works immediately on launch.

    **Step 6: Camera controller** — Create `apollo7/rendering/camera.py` with `CameraController` class wrapping `gfx.OrbitController`. Configure:
    - Default three-quarter view angle (azimuth ~45 degrees, elevation ~30 degrees)
    - Auto-frame method that positions camera to show full scene
    - Orbit via left-click-drag, zoom via scroll wheel, pan via right-click-drag (OrbitController defaults)

    **Step 7: Main window** — Create `apollo7/gui/main_window.py` with `MainWindow(QMainWindow)`:
    - Window title "Apollo 7", default size 1920x1080, minimum 1280x720
    - Horizontal QSplitter: left side (viewport, ~73%) | right side (placeholder panels, ~27%)
    - Left side has vertical QSplitter: viewport on top (~85%) | bottom strip placeholder (~15%, collapsible)
    - Right side has vertical QSplitter: controls placeholder on top | library placeholder on bottom
    - Placeholder panels are QWidget with QLabel showing panel name (e.g., "Controls", "Library", "Features") — these get replaced in later plans
    - Apply theme QSS to QApplication in app.py

    **Step 8: Test infrastructure** — Create:
    - `tests/__init__.py` (empty)
    - `tests/conftest.py` with fixtures: `tmp_dir` (temp directory), `sample_image` (generates a 100x100 RGB numpy array with gradient colors and saves as JPEG to tmp_dir)
    - `tests/test_gpu_providers.py` with smoke test: import onnxruntime, check that `get_available_providers()` returns a list (don't assert DirectML — may not be available in CI), and a test that imports pygfx and wgpu without error

    **IMPORTANT constraints:**
    - No CUDA dependencies anywhere. wgpu uses Vulkan/DX12, ONNX uses DirectML.
    - Use `rendercanvas.qt.QRenderWidget` — NOT manual surface creation
    - Do NOT use `gfx.OrbitController` `register_events` if it doesn't exist — check the API. The research example shows `register_events=self._renderer` but verify pygfx 0.16 API.
    - Ensure `__main__.py` calls `sys.exit(app.exec())` for clean shutdown
