  <action>
    **Depth extractor** — Create `apollo7/extraction/depth.py` with `DepthExtractor(BaseExtractor)`:
    - `name = "depth"`
    - Constructor takes `model_path: str` (default: "models/depth_anything_v2_vits.onnx")
    - Lazy model loading: ONNX session created on first `extract()` call, not at import time
    - Follow the research code example exactly for preprocessing: resize to model input size, ImageNet normalization, HWC->NCHW
    - Check providers at session creation: log warning if DmlExecutionProvider is not available (CPU fallback). Don't crash — just warn.
    - Post-process: squeeze output, resize to original image dimensions via bilinear interpolation, normalize to [0,1]
    - Returns ExtractionResult with `data={"min_depth": float, "max_depth": float}`, `arrays={"depth_map": np.ndarray}`
    - Handle missing model file gracefully: raise FileNotFoundError with message about downloading the model

    **Extraction pipeline** — Create `apollo7/extraction/pipeline.py` with `ExtractionPipeline`:
    - Takes list of BaseExtractor instances
    - `run(image: np.ndarray, photo_path: str, cache: FeatureCache | None = None) -> dict[str, ExtractionResult]`
    - For each extractor: check cache first, if cached return cached result, otherwise extract and store in cache
    - Emits per-extractor progress (useful for progress bar)
    - Extraction order follows configured order (color first per user decision)

    **Depth-projected point cloud** — Create `apollo7/pointcloud/depth_projection.py`:
    - `generate_depth_projected_cloud(image, depth_map, depth_exaggeration=4.0, layer_offset=0.0) -> tuple[np.ndarray, np.ndarray, np.ndarray]`
    - Follow research code example: create x,y grid normalized to [-1,1], z from depth * exaggeration
    - `layer_offset` parameter for stacked layer mode (each photo offset in Z)
    - Original pixel colors with alpha=1.0
    - Uniform point sizes (default 2.0 from settings)
    - Full pixel density: every pixel becomes a point

    **Feature-clustered point cloud** — Create `apollo7/pointcloud/feature_cluster.py`:
    - `generate_feature_clustered_cloud(image, features: dict[str, ExtractionResult]) -> tuple[np.ndarray, np.ndarray, np.ndarray]`
    - Use color-based clustering: group pixels by dominant color similarity, position groups in 3D space
    - Algorithm: downsample image for clustering (e.g., every 4th pixel), KMeans or color distance grouping, assign 3D positions per cluster with scatter within cluster
    - Points retain original pixel colors
    - This mode is more abstract/artistic — floating clusters in space

    **LOD system** — Create `apollo7/pointcloud/lod.py`:
    - `decimate_points(positions, colors, sizes, factor: float) -> tuple[np.ndarray, np.ndarray, np.ndarray]`
    - Grid-based spatial decimation: divide space into cells, keep one point per cell (the one closest to cell center)
    - `factor` controls cell size (0.5 = half the points, 0.25 = quarter)
    - Preserves spatial distribution and color representation
    - Fast: must handle 12M points without excessive delay

    **Generator facade** — Create `apollo7/pointcloud/generator.py` with `PointCloudGenerator`:
    - `generate(image, features, mode="depth_projected", **kwargs) -> tuple[np.ndarray, np.ndarray, np.ndarray]`
    - Delegates to depth_projection or feature_cluster based on mode
    - Applies LOD if point count exceeds budget (default 5M rendering budget)
    - Returns (positions, colors, sizes) ready for viewport

    **IMPORTANT constraints:**
    - Color space: input images are RGB float32 [0,1]. DepthExtractor must convert to uint8 for ONNX input preprocessing.
    - Depth map resizing must use bilinear interpolation to avoid blocky artifacts
    - Point cloud arrays must be contiguous float32 for GPU buffer upload
    - For unit tests: mock ONNX session to avoid requiring model file. Test the pre/post-processing logic, not the model inference.
