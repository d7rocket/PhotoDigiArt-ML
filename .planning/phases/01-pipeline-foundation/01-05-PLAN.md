  <action>
    **Extraction worker** — Create/update `apollo7/workers/extraction_worker.py` with `ExtractionWorker(QRunnable)`:
    - Takes: list of photo paths, extraction pipeline, point cloud generator, feature cache
    - WorkerSignals: `progress(int, int)` (current, total), `photo_complete(str, dict, object)` (path, features, point_cloud_data), `finished()`, `error(str)`
    - For each photo: run extraction pipeline (color, edges, depth in order), generate point cloud, emit photo_complete with both features and point cloud arrays
    - Per user decision: extraction order is color first, then edges, then depth
    - Handle errors per-photo (skip and continue, emit error signal)

    **Progressive viewport build** — Update `apollo7/gui/widgets/viewport_widget.py`:
    - Method `add_photo_cloud(photo_id: str, positions, colors, sizes, layer_index: int)`: adds a point cloud to the scene tagged with photo_id. For stacked layers mode, apply Z-offset based on layer_index.
    - Method `remove_photo_cloud(photo_id: str)`: removes a specific photo's points
    - Method `set_layout_mode(mode: str)`: switches between "depth_projected" and "feature_clustered" — clears and regenerates all point clouds (triggers re-generation via signal)
    - Method `set_multi_photo_mode(mode: str)`: "stacked" or "merged" — adjusts Z-offsets or merges
    - Method `update_point_material(point_size=None, opacity=None)`: updates material properties on all point objects in scene
    - Auto-frame after each photo is added (calls camera.show_object)
    - Remove the test point sphere from Plan 01 — replace with real data

    **Camera improvements** — Update `apollo7/rendering/camera.py`:
    - Default three-quarter view: after auto-frame, set azimuth ~45 degrees, elevation ~30 degrees (per user decision)
    - Ensure auto-frame works correctly with growing point clouds (not just first load)

    **Controls panel wiring** — Update `apollo7/gui/panels/controls_panel.py`:
    - "Extract Features" button: triggers extraction on all loaded photos (or selected photos)
    - Layout mode toggle: Depth-projected (default) / Feature-clustered radio buttons — switching triggers re-generation
    - Multi-photo mode toggle: Stacked layers (default) / Merged cloud
    - Point size slider (range from settings: 0.5 to 10.0, default 2.0)
    - Opacity slider (0.0 to 1.0, default 1.0)
    - Depth exaggeration slider (1.0 to 10.0, default 4.0) — changing this triggers re-generation
    - All sliders update viewport in real-time via signals
    - Re-extract button: re-runs extraction on selected photo(s) clearing cache first

    **Main window wiring** — Update `apollo7/gui/main_window.py`:
    - Connect "Extract Features" button to create ExtractionWorker, start in QThreadPool
    - Connect ExtractionWorker.photo_complete to:
      1. Add point cloud to viewport (progressive build — sculpture grows as photos complete)
      2. Update feature strip with extraction results for latest photo
      3. Update library panel with extraction status
    - Connect ExtractionWorker.progress to progress bar
    - Connect ExtractionWorker.finished to progress bar finish + auto-frame camera
    - Connect controls panel sliders to viewport update methods
    - Connect layout mode toggle to viewport set_layout_mode
    - Connect library panel photo_selected to feature strip update
    - Store extraction results and point cloud data per photo for re-generation on mode switch

    **IMPORTANT constraints:**
    - All extraction and point cloud generation runs in worker threads, NEVER main thread
    - Point cloud arrays (positions, colors, sizes) are generated in worker thread but added to scene in main thread (pygfx scene modification must be on main thread)
    - Per user decision: progressive viewport build — point cloud grows in real-time as each photo completes
    - Per user decision: stacked layers is default multi-photo mode
    - Per user decision: depth exaggeration 3-5x default (use 4.0)
    - Per user decision: re-extractable — user can re-run extraction with different settings
    - LOD applied automatically when total point count exceeds 5M rendering budget
