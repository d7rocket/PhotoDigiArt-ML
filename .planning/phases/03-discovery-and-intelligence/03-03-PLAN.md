- MappingConnection dataclass:
  - source_feature: str (e.g., "semantic", "color", "depth", "edge")
  - source_key: str (e.g., "mood_tags.serene", "dominant_colors.0.saturation", "depth_mean", "edge_density")
  - target_param: str (SimulationParams field name, e.g., "speed", "turbulence_scale")

  - to_dict() / from_dict() for persistence

Create apollo7/mapping/engine.py:
- Define FEATURE_SOURCES dict mapping (feature_name, key) -> description for UI labels:
  - ("semantic", "mood_tags.serene") -> "Mood: Serene"
  - ("semantic", "mood_tags.chaotic") -> "Mood: Chaotic"
  - ("semantic", "mood_tags.joyful") -> "Mood: Joyful"
  - ("color", "dominant_saturation") -> "Color: Avg Saturation"
  - ("color", "dominant_brightness") -> "Color: Avg Brightness"
  - ("depth", "depth_mean") -> "Depth: Mean"
  - ("depth", "depth_range") -> "Depth: Range"
  - ("edge", "edge_density") -> "Edge: Density"
- Define TARGET_PARAMS list of tunable SimulationParams field names with display labels and ranges
- MappingEngine:
  - extract_feature_value(feature_data: dict[str, ExtractionResult], source_feature: str, source_key: str) -> float | None
    - Navigate into ExtractionResult data/arrays to get a scalar value
    - For mood_tags/object_tags: look up the label's confidence score
    - For arrays: compute summary stats (mean, range, density)
    - Normalize output to [0, 1] range
  - evaluate(graph: MappingGraph, feature_data: dict[str, ExtractionResult]) -> dict[str, float]
    - For each connection: get feature value, multiply by strength, add to target accumulator
    - Return dict mapping target_param -> accumulated value
    - Skip connections where feature value is None (missing data)

Create apollo7/gui/widgets/node_editor.py:
- Port(QGraphicsEllipseItem):
  - RADIUS = 6, colored circle
  - is_output: bool (True = feature source on left, False = param target on right)
  - name: str (the feature key or param name)
  - node: reference to parent NodeItem
  - Visual: filled circle, hover highlight

- NodeItem(QGraphicsRectItem):
  - Fixed position (not movable -- feature nodes on left column, param nodes on right)
  - label: str displayed in header
  - ports: list[Port] arranged vertically
  - Visual: rounded rect with dark background (#1E1E1E), label in accent color (#0078FF for features, #00CC66 for params)
  - Width: 160px, port spacing: 24px

- Wire(QGraphicsPathItem):
  - Cubic Bezier curve from source port to target port (QPainterPath.cubicTo with horizontal control points at 1/3 distance)
  - strength: float shown as small label at midpoint
  - Color-coded by feature type (warm=semantic, cool=color, grey=depth/edge)
  - Click to select, right-click to delete
  - Selected wire shows strength slider popup (QDoubleSpinBox or small slider)

- PatchBayScene(QGraphicsScene):
  - Left column: feature output nodes (Semantic Mood, Semantic Objects, Color, Depth, Edge)
  - Right column: parameter input nodes (all SimulationParams tunable fields)
  - Wire creation: drag from output port, temporary wire follows mouse, release on input port creates MappingConnection
  - Signal: connection_added(MappingConnection), connection_removed(MappingConnection), strength_changed(MappingConnection)
  - set_graph(MappingGraph) to load existing connections as wires
  - get_graph() -> MappingGraph to export current state

- PatchBayEditor(QWidget):
  - Contains QGraphicsView with the PatchBayScene
  - Header bar with title "Feature Mapping" and close button (X)
  - Footer with "Clear All" button
  - Designed to be shown as a full overlay/modal in MainWindow
  - Keyboard shortcut: Escape to close
  - Signal: mapping_changed(MappingGraph) emitted on any connection change

Styling:
- Scene background: #1A1A1A (darker than panels)
- Grid lines: subtle #222222 dots
- Wire colors: use Bezier curves, not straight lines (per research pitfall #4)
- Ports glow on hover
- Overall aesthetic: DAW routing matrix feel -- professional, clean, functional

Note: Integration with MainWindow (showing/hiding overlay, wiring signals) will happen in a later integration plan. This task creates the self-contained widget.
