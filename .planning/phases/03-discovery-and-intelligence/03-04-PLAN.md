    - DBSCAN with metric="cosine", eps=0.3, min_samples=2 identifies natural clusters
    - UMAP n_components=3 projects 512-dim embeddings to 3D positions
    - n_neighbors clamped to min(15, n_samples-1) to handle small collections
    - Collections under 3 photos skip UMAP, use simple line/triangle placement
    - CollectionResult contains paths, labels, positions_3d, cluster_centroids
    - Cluster centroids computed as mean of member embeddings (in original space)
    - Force attractor data: list of (position_3d, weight) tuples for simulation integration
  </behavior>

Create apollo7/collection/__init__.py with module docstring.

Create apollo7/collection/analyzer.py:
- CollectionResult dataclass:
  - paths: list[str] -- photo paths in order
  - labels: np.ndarray -- DBSCAN cluster labels (-1 = noise/outlier)
  - positions_3d: np.ndarray -- (N, 3) float32 UMAP projections
  - cluster_centroids: dict[int, np.ndarray] -- cluster_id -> centroid in original 512-dim space
  - cluster_positions_3d: dict[int, np.ndarray] -- cluster_id -> centroid in 3D UMAP space
  - n_clusters: int (excluding noise label -1)

- CollectionAnalyzer:
  - analyze(embeddings: dict[str, np.ndarray]) -> CollectionResult:
    - Stack embeddings into (N, 512) matrix
    - Run DBSCAN(eps=0.3, min_samples=2, metric="cosine")
    - Run UMAP(n_components=3, n_neighbors=min(15, N-1), min_dist=0.1, random_state=42)
    - Handle N < 3: place on line (N=1: origin, N=2: [-1,0,0] and [1,0,0])
    - Compute cluster centroids in both original and 3D space
    - Scale UMAP output to [-5, 5] range (per research recommendation)
  - get_force_attractors(result: CollectionResult) -> list[tuple[np.ndarray, float]]:
    - For each cluster (excluding noise): return (3D centroid position, cluster_size/total_size as weight)
    - Outlier photos treated equally -- no special treatment (per user decision)

- test_umap_projection_shape: positions_3d is (N, 3)
- test_umap_small_collection: 2 embeddings don't crash, use fallback placement
- test_dbscan_labels: labels array has length N with values >= -1
- test_cluster_centroids: centroids are mean of member embeddings
- test_force_attractors: returns list of (position, weight) tuples, weights sum to <= 1.0
- test_umap_scale: output positions are within [-6, 6] range (approximately [-5, 5] with some margin)
- test_single_photo: single embedding handled gracefully

Create apollo7/collection/embedding_cloud.py:
- CLUSTER_PALETTE: list of 10 distinct colors for cluster visualization (avoid red/green for accessibility)
- create_embedding_cloud(result: CollectionResult) -> pygfx.Points:
  - Create pygfx.Geometry with positions_3d, vertex colors by cluster label
  - Noise points (-1 label): neutral gray (0.5, 0.5, 0.5, 0.6)
  - Cluster points: color from CLUSTER_PALETTE based on label
  - Point size: 8.0 for photo points (larger than sculpture particles)
  - Material: PointsMaterial with color_mode="vertex", size_mode="vertex"
- create_cluster_labels(result: CollectionResult) -> list[pygfx.Text]:
  - Small text labels at cluster centroid positions showing cluster ID or dominant mood
  - Optional: only create if cluster has 3+ members

- EmbeddingCloudManager:
  - __init__(viewport): stores reference to viewport widget
  - _cloud_points: pygfx.Points | None
  - _visible: bool
  - update(result: CollectionResult): remove old cloud, create new, add to scene
  - toggle_visibility(): show/hide the embedding cloud
  - isolate_cluster(cluster_id: int, result: CollectionResult):
    - Dim all sculpture particles NOT belonging to photos in this cluster
    - Set alpha of non-cluster embedding points to 0.15
    - Set alpha of cluster embedding points to 1.0
    - Emit signal with list of photo paths in the cluster
  - clear_isolation(): restore all particles and embedding points to normal alpha
  - Signals: cluster_isolated(list[str]), isolation_cleared()

  - Look up cluster label for that point
  - Call EmbeddingCloudManager.isolate_cluster()
  - Second click on same cluster or empty space clears isolation
- Add toggle_embedding_cloud() method connected to a keyboard shortcut or button

Update apollo7/simulation/engine.py:
- Add set_attractors(attractors: list[tuple[np.ndarray, float]]) method:
  - Store attractor positions and weights
  - Create a GPU buffer for attractor data (position xyz + weight, packed as vec4 per attractor)
  - Maximum 16 attractors (pad with zeros if fewer)
  - Upload to a new bind group slot or extend existing uniform buffer
- Modify the forces compute shader (or add a new attractor pass):
  - For each particle, compute attraction toward each attractor position
  - Force = weight * (attractor_pos - particle_pos) / distance^2 (with softening to avoid singularity)
  - Add attractor forces to the force accumulation buffer
  - This integrates with the existing multi-pass pipeline: forces -> SPH -> integration
- Add clear_attractors() to remove all attractor forces

Update apollo7/simulation/parameters.py:
- Add attractor_global_strength: float = 0.5 to SimulationParams
  - This scales all attractor forces globally
  - Add to _visual_params set for hot-reload
  - Add to to_uniform_bytes() packing (use the _pad2 slot in vec4[6])

Wire collection analysis results:
