# Phase 6 -- UI Design Contract

> Visual and interaction contract for Phase 6: Interface and Intelligence. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | PySide6 QSS + qt-material dark theme |
| Preset | qt-material dark, custom QSS overrides |
| Component library | PySide6 (QTabWidget, QGroupBox, QSlider, QPushButton) |
| Icon library | None (text labels, gradient-generated preset thumbnails) |
| Font | Segoe UI, sans-serif (established in theme.py) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, radio/checkbox spacing |
| sm | 8px | Panel content margins, slider row spacing, button gaps |
| md | 16px | Button horizontal padding, tab padding, group box content |
| lg | 24px | Section breaks within a tab, card padding |
| xl | 32px | Tab content top margin, major section separators |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Preset thumbnail grid cells use 80x60px fixed size for gradient icons. Toolbar strip height is 44px (touch-friendly).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 (regular) | 1.5 |
| Label | 11px | 400 (regular) | 1.4 |
| Heading | 15px | 600 (semibold) | 1.2 |
| Display | 16px | 600 (semibold) | 1.2 |

Notes:
- Body at 13px matches existing theme.py base font-size.
- Label at 11px used for slider value readouts, FPS counter, and parameter chips on Claude suggestion cards.
- Heading at 15px used for panel-title (existing `#panel-title` QSS), section headers inside tabs, and collapsible group headers. The 2px gap from Body (13px) ensures a visible size distinction at screen resolution.
- Display at 16px used only for the Claude suggestion card artistic rationale text.

## Interaction Contract

### Tab Switching
- Instant switch, no animation (standard QTabWidget behavior)
- Tab content scroll position preserved per tab

### Slider Interaction
- All sliders emit signals on value change (existing pattern)
- Cohesion slider uses crossfade interpolation over 400ms (existing CrossfadeEngine)
- Mouse wheel blocked on sliders (existing eventFilter)

### Claude Analysis Flow
1. User clicks "Analyze with Claude" -- button becomes disabled, spinner shows
2. Background worker calls Claude API (existing QRunnable pattern)
3. On success: suggestion card appears with rationale + chips + Apply button
4. On error: error state shown in card area (see Copywriting)
5. User clicks Apply -- parameters crossfade via CrossfadeEngine (400ms)
6. Direction buttons appear after Apply
7. Clicking direction button: sends refinement request, spinner shows, new card replaces old
8. "Keep this" dismisses direction buttons, preserves current params
9. "Start over" clears suggestion card, returns to initial Analyze button state

### Preset Application
- Click preset card -- parameters apply via CrossfadeEngine (400ms crossfade)
- Selected card gets accent border
- Only one preset active at a time

### Undo Integration
- Claude Apply wraps in ParameterChangeCommand (existing undo pattern)
- Preset Apply wraps in ParameterChangeCommand
- Ctrl+Z reverts to pre-apply state

## State Transitions

### Claude Panel States

```
[idle] --click Analyze--> [loading] --success--> [suggestion]
                                   --error--> [error]
[suggestion] --click Apply--> [applied] --click direction--> [loading]
                                       --click Keep This--> [idle]
                                       --click Start Over--> [idle]
[error] --click Retry--> [loading]
```

### Preset Panel States

```
[grid visible] --click card--> [crossfade active, card selected]
                              --crossfade complete--> [card selected, params applied]
```
