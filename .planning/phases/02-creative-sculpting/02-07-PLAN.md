   - `_sph_forces_buf`: storage buffer, size = max_particles * 16 bytes. Usage: STORAGE | COPY_DST.
   - `_densities_buf`: storage buffer, size = max_particles * 4 bytes (f32 per particle). Usage: STORAGE | COPY_DST.
   - `_cell_counts_buf`: storage buffer, size = GRID_SIZE^3 * 4 bytes. Usage: STORAGE | COPY_DST.
   - `_cell_offsets_buf`: storage buffer, size = GRID_SIZE^3 * 4 bytes. Usage: STORAGE | COPY_DST.

2. Add a `build_spatial_hash(self, positions: np.ndarray)` method that:
   - Takes (N, 3) float32 positions
   - Computes cell index for each particle: cell = floor((pos + 64.0) / cell_size) with cell_size = repulsion_radius * 2.0 (pass cell_size as parameter, default 0.2)
   - Clamps cell indices to [0, 127]
   - Counts particles per cell (cell_counts array)
   - Computes prefix sum for cell_offsets
   - Creates sorted_indices array (particles sorted by cell)
   - Uploads all three arrays to their GPU buffers via device.queue.write_buffer()

   NOTE: This is a CPU-side spatial hash build. It only needs to run once at simulation start and on restart (not per-frame). The forces shader reads these buffers. For millions of particles, a GPU-side hash build would be better, but CPU is acceptable for the initial implementation since it only runs on init/restart, not per frame.

   - `cell_counts_buffer` -> `_cell_counts_buf`
   - `cell_offsets_buffer` -> `_cell_offsets_buf`
   - `sorted_indices_buffer` -> `_sorted_indices_buf`

4. In upload(), after uploading particle state, call build_spatial_hash(positions) with default cell_size.

5. Clear forces buffer to zeros on each step by adding a `clear_forces(self)` method that writes zeros to _forces_buf and _sph_forces_buf via device.queue.write_buffer.

6. **Modify `_step_once()`** to dispatch all passes in order:
   ```
   a. Clear forces buffers (particle_buffer.clear_forces())
   b. Rebuild all bind groups (forces, SPH, integrate)
   c. Create command encoder
   d. Begin compute pass
   e. Dispatch forces pipeline (chunked) -- always runs
   f. End compute pass, submit, start new encoder+pass
   g. If NOT performance_mode:
      - Dispatch SPH density pipeline (chunked)
      - End pass, submit, start new encoder+pass
      - Dispatch SPH force pipeline (chunked)
      - End pass, submit
   h. Start new encoder+pass
   i. Dispatch integration pipeline (chunked)
   j. End pass, submit
   k. Swap buffers
   ```

   IMPORTANT: Each pipeline dispatch must be in its own command encoder submission to ensure proper synchronization between passes (forces must complete before SPH reads, SPH must complete before integration reads). This is because wgpu does not guarantee execution order within a single compute pass for different pipelines reading/writing shared buffers.
