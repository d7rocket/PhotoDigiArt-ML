  <behavior>
    - ParameterChangeCommand stores param name, old value, new value, and apply callback
    - redo() calls apply_fn(param_name, new_value)
    - undo() calls apply_fn(param_name, old_value)
    - mergeWith() collapses consecutive same-parameter changes (same merge ID) into one entry
    - Two commands with different param names do NOT merge
    - QUndoStack integration: Ctrl+Z triggers undo, Ctrl+Shift+Z triggers redo
    - After 5 rapid slider changes on same param, undo stack has only 1 entry
  </behavior>
  <action>
**undo_commands.py:**
Create `ParameterChangeCommand(QUndoCommand)` per the pattern in RESEARCH.md:
- Constructor: `(param_name: str, old_value: float, new_value: float, apply_fn: Callable, merge_id_offset: int = 0)`
- `id()` returns `_BASE_MERGE_ID + merge_id_offset` (each unique param gets a unique offset so only same-param changes merge)
- `mergeWith(other)`: if same id, update self._new to other._new, return True
- `redo()`: call apply_fn(param_name, new_value)
- `undo()`: call apply_fn(param_name, old_value)

Also create `ResetSectionCommand(QUndoCommand)`:
- Stores a dict of {param_name: (old_value, default_value)} for all params in a section
- redo() applies all defaults, undo() restores all old values
- Does NOT merge (id() returns -1)

**main_window.py modifications:**
- Add `self._undo_stack = QUndoStack(self)` in __init__
- Create undo/redo QActions from undo_stack with shortcuts:
  - `undo_action = self._undo_stack.createUndoAction(self, "Undo")` with `Qt.CTRL | Qt.Key_Z`
  - `redo_action = self._undo_stack.createRedoAction(self, "Redo")` with `Qt.CTRL | Qt.SHIFT | Qt.Key_Z`
  - `self.addAction(undo_action)` and `self.addAction(redo_action)`
- Wrap existing slider signal connections to push ParameterChangeCommand instead of directly applying:
  - Track "previous value" per slider to compute old_value for each command
  - The apply_fn for point_size calls `self.viewport.update_point_material(point_size=value)`
  - The apply_fn for opacity calls `self.viewport.update_point_material(opacity=value)`
  - The apply_fn for depth_exag calls `self._on_depth_exaggeration_changed(value)`
- Assign unique merge_id_offsets: point_size=0, opacity=1, depth_exaggeration=2 (extend in Plan 03 for sim params)

**Tests** (`tests/test_undo_redo.py`):
- Test ParameterChangeCommand redo/undo calls apply_fn correctly
- Test mergeWith collapses same-id commands
- Test mergeWith rejects different-id commands
- Test QUndoStack with 5 pushes of same param -> 1 undo reverts to original value
- Test QUndoStack with mixed params -> each param gets its own undo entry
