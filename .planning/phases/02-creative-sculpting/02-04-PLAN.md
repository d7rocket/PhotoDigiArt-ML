**BloomController** (`bloom.py`):
Thin wrapper around pygfx's `PhysicalBasedBloomPass`:
- `__init__(self, renderer: gfx.WgpuRenderer)`: create PhysicalBasedBloomPass with default strength=0.4, add to renderer.effect_passes.
- `set_strength(self, value: float)`: update bloom_strength (range 0.0-3.0).
- `set_enabled(self, enabled: bool)`: toggle bloom_pass.enabled.
- `@property strength`, `@property enabled`.

**DepthOfFieldPass** (`dof_pass.py`):
Custom post-processing effect. Approach options (Claude's discretion on implementation):
- **Option A (recommended)**: Custom fragment shader that reads depth buffer and applies Gaussian blur based on distance from focal plane. Subclass or mimic the pattern from pygfx's PhysicalBasedBloomPass source. Read the pygfx source to understand the EffectPass contract.
- **Option B (fallback)**: If custom EffectPass subclassing proves too difficult (Open Question 2 in RESEARCH.md), implement as a simple screen-space blur using wgpu compute shader that reads the rendered framebuffer and applies circle-of-confusion blur. This is independent of pygfx's effect pass system.
- Parameters: focal_distance (0.0-50.0, default 10.0), aperture (0.1-5.0, default 2.0, controls blur intensity), enabled toggle.
- If EffectPass subclassing is not feasible, document why and implement the compute-shader fallback. The feature must ship.

**SSAOPass** (`ssao_pass.py`):
Screen-space ambient occlusion:
- Same approach decision as DoF -- try pygfx EffectPass first, fall back to compute shader.
- Parameters: radius (0.1-2.0, default 0.5), intensity (0.0-2.0, default 1.0), enabled toggle.
- SSAO needs depth buffer access. If pygfx doesn't expose depth buffer to effect passes, skip SSAO and document as a known limitation. Bloom and DoF are higher priority.

**TrailAccumulator** (`trails.py`):
Alpha trail effect for showing particle motion history:
- Maintains an accumulation texture (same size as viewport).
- Each frame: render current particles at full alpha on top of the previous frame faded by a decay factor (0.9-0.99, configurable).
- Implementation: custom render pass that blends current frame onto accumulation buffer with alpha = trail_decay.
- If pygfx effect pass system doesn't support this cleanly, implement as a separate render-to-texture step in the _animate() callback:
  1. Render scene to offscreen texture
  2. Blend offscreen texture onto accumulation texture (multiply accumulation by decay, add current frame)
  3. Display accumulation texture
- Parameters: trail_length (0.0-1.0 mapped to decay 0.0-0.99, default 0.5), enabled toggle.
- NOTE: This may require significant experimentation. If clean integration proves too complex, implement a simpler version: spawn "ghost" particles at previous positions with decaying alpha, rendered as additional point cloud objects. This is less elegant but ships.
