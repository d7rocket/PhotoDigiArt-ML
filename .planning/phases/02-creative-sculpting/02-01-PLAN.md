**SimulationParams dataclass** (`parameters.py`):
- All tunable parameters as a frozen-capable dataclass: noise_frequency (default 0.5), noise_amplitude (default 1.0), noise_octaves (default 4), turbulence_scale (default 1.0), viscosity (default 0.1), pressure_strength (default 1.0), surface_tension (default 0.01), attraction_strength (default 0.5), repulsion_strength (default 0.3), repulsion_radius (default 0.1), gravity (default vec3(0,-0.1,0)), wind (default vec3(0,0,0)), speed (default 1.0), dt (default 0.016), damping (default 0.99), smoothing_radius (default 0.1), rest_density (default 1000.0), gas_constant (default 2.0).
- Categorize params: visual_params (hot-reload list) vs physics_params (restart list) as class-level sets.
- `to_uniform_bytes()` method: pack into a flat float32 array matching WGSL struct layout. Use vec4 padding everywhere (WGSL alignment -- see Pitfall 1 in RESEARCH.md). Total uniform struct size must be a multiple of 16 bytes.
- `is_visual_param(name)` and `is_physics_param(name)` class methods.

**ParticleBuffer** (`buffers.py`):
- Manages double-buffered GPU storage for particle state. Each particle = 2x vec4 (position.xyz + life_w, velocity.xyz + mass_w) = 32 bytes.
- `__init__(self, device: wgpu.GPUDevice, max_particles: int)`: create buf_a and buf_b with STORAGE | COPY_SRC | COPY_DST usage flags.
- `upload(self, positions: np.ndarray, colors: np.ndarray)`: pack initial state into buf_a. Colors stored in a separate buffer (4x float32 RGBA per particle).
- `swap(self)`: swap buf_a and buf_b references (just pointer swap, no GPU copy).
- `read_positions(self) -> np.ndarray`: read back positions from current output buffer (for export/save only -- NOT per-frame).
- Properties: `input_buffer`, `output_buffer`, `color_buffer`, `particle_count`.
- Create a separate uniform buffer for SimulationParams, updated via `update_params(params: SimulationParams)`.

**WGSL Shaders:**

`noise.wgsl`: Port Perlin 3D noise and simplex 3D noise from munrocket/wgsl-noise gist. Functions: `fn perlin3d(p: vec3<f32>) -> f32`, `fn simplex3d(p: vec3<f32>) -> f32`, `fn fbm3d(p: vec3<f32>, octaves: u32) -> f32` (fractal Brownian motion using Perlin).

`integrate.wgsl`: Final integration pass. Reads all force contributions (flow field, external forces, SPH forces). Symplectic Euler integration: vel += (forces * dt), vel *= damping, pos += vel * dt. Boundary clamping to prevent particles escaping to infinity (soft bounce at boundaries). Write to output particle buffer.

**Module init** (`__init__.py`):
- Export SimulationParams, ParticleBuffer from submodules.

**Tests** (`tests/test_simulation_params.py`):

CRITICAL alignment rules for WGSL structs:
- Use vec4 everywhere, never bare vec3 (vec3 aligns to 16 bytes but only occupies 12, causing silent corruption)

**Shader loading** (`shaders/__init__.py`):
- `load_shader(name: str) -> str`: read .wgsl file from package directory using `importlib.resources` or `pathlib.Path(__file__).parent`.
- `build_combined_shader(*names) -> str`: concatenate multiple .wgsl files into a single shader module (noise first, then others that depend on it).

**WGSL Shaders:**

`flow_field.wgsl`: Feature-driven flow field computation. Samples edge_map and depth_map textures (bound as texture_2d). Maps 3D particle position to 2D UV coords. Edge intensity drives turbulence magnitude, depth drives current direction. Combines with Perlin noise for organic flow. Main function: `fn compute_flow(pos: vec3<f32>, time: f32, params: SimParams) -> vec3<f32>`.

`forces.wgsl`: Attraction/repulsion (simple pairwise within a radius -- use spatial hash grid for efficiency), gravity (uniform directional), wind (uniform directional with optional turbulence modulation). Main function: `fn compute_forces(pos: vec3<f32>, vel: vec3<f32>, params: SimParams) -> vec3<f32>`.

`sph.wgsl`: Three-pass SPH implementation:
1. Density pass: compute density at each particle using poly6 kernel, spatial hash for neighbor lookup.
2. Force pass: compute pressure force (Spiky kernel gradient) + viscosity force (viscosity kernel laplacian).
3. Results written to intermediate buffer consumed by integration pass.
Spatial hashing: divide space into cells of size = smoothing_radius. Sort particles into cells. Only search 27 adjacent cells (3x3x3). Use atomic operations for cell counts.

**Compute pipeline setup:**
- Create separate compute pipelines for each pass (flow, forces, SPH density, SPH forces, integration).
- Each pipeline has its own bind group layout and bind group.
- Shared resources: particle buffers (input/output), params uniform, time uniform.
- Feature textures: bound as texture_2d with samplers in flow field pipeline.
- SPH spatial hash: allocate grid buffers (cell_counts, cell_offsets, particle_indices) sized for the simulation domain.

**Feature texture upload:**
- Edge maps and depth maps uploaded as R32Float textures.
- Color distribution could be uploaded as a small texture or uniform array.
- Textures created once on initialize(), not per-frame.

**Chunked dispatch pattern (critical for AMD):**
```python
def _dispatch_chunked(self, pass_encoder, pipeline, bind_group, total_particles):
    CHUNK = 256 * 1024  # 256K particles per dispatch
    for offset in range(0, total_particles, CHUNK):
        count = min(CHUNK, total_particles - offset)
        workgroups = (count + 255) // 256
        # Update offset uniform
        pass_encoder.set_pipeline(pipeline)
        pass_encoder.set_bind_group(0, bind_group)
        pass_encoder.dispatch_workgroups(workgroups)
```

**Performance mode support:**
- `set_performance_mode(enabled: bool)`: when True, skip SPH pass and reduce sim steps.
- `set_sim_steps_per_frame(n: int)`: default 1, can increase for faster sim or decrease for performance.
